# Historias de Usuario — BC Alerting

Esta sección presenta las Épicas e Historias de Usuario identificadas para el bounded context de **Alerting (Alertas)** de la plataforma Clair IoT que se encuentran actualmente implementadas en el sistema. Cada historia está redactada desde la perspectiva de un rol real de la plataforma e incluye criterios de aceptación detallados que describen el flujo de negocio y comportamiento lógico sin hacer referencia a elementos específicos de la interfaz de usuario.

---

## Epics

| Epic / Story ID | Título | Descripción | Criterios de Aceptación | Relacionado con (Epic ID) |
|---|---|---|---|---|
| EPIC-01 | Visibilidad de Alertas Activas | Como Usuario, quiero consultar las alertas activas y reconocidas de mis dispositivos con indicación de su gravedad, para poder reaccionar oportunamente ante anomalías ambientales. | La Épica se completa cuando todas las historias de usuario relacionadas han sido implementadas, validadas y aceptadas. | - |
| EPIC-02 | Consulta de Historial de Alertas | Como Usuario, quiero consultar el registro histórico de las alertas que ya han sido resueltas en el sistema, para realizar un análisis de incidentes pasados de calidad del aire. | La Épica se completa cuando todas las historias de usuario relacionadas han sido implementadas, validadas y aceptadas. | - |
| EPIC-03 | Resumen Diario de Alertas | Como Usuario, quiero obtener un consolidado del volumen diario de alertas registradas a lo largo del tiempo, para identificar patrones de comportamiento y picos en la frecuencia de las alertas. | La Épica se completa cuando todas las historias de usuario relacionadas han sido implementadas, validadas y aceptadas. | - |

## User Stories

| Epic / Story ID | Título | Descripción | Criterios de Aceptación | Relacionado con (Epic ID) |
|---|---|---|---|---|
| US-01 | Ver Alertas Activas | Como Usuario, quiero consultar las alertas que se encuentran activas en la plataforma, para identificar qué dispositivos están superando actualmente los límites de calidad del aire configurados. | Dado que el Usuario solicita la información de alertas activas, cuando el sistema procesa la consulta, entonces devuelve los registros de alerta cuyo estado sea ACTIVE o ACKNOWLEDGED, incluyendo el ID de la alerta, el nombre del dispositivo, el nombre del espacio, la métrica involucrada, el umbral de alerta configurado, el valor medido que disparó la alerta, el mensaje explicativo, la severidad, el estado y la fecha de ocurrencia. Dado que no existen alertas activas en el sistema, cuando se realiza la consulta, entonces el sistema devuelve un resultado indicando que no hay alertas pendientes de atención. | EPIC-01 |
| US-02 | Identificar la Gravedad de la Alerta | Como Usuario, quiero conocer el nivel de gravedad de las alertas del sistema, para priorizar mi atención en las anomalías más críticas de la calidad del aire. | Dado que el Usuario consulta las alertas de la plataforma, cuando el sistema evalúa su severidad, entonces clasifica cada alerta en los niveles CRITICAL, WARNING o LOW. Las alertas categorizadas como CRITICAL son marcadas para priorizar su atención sobre las de severidad WARNING y LOW. | EPIC-01 |
| US-03 | Navegar por la Lista de Alertas | Como Usuario, quiero paginar los registros de la lista de alertas, para poder revisar todos los eventos cuando el volumen total supere la capacidad de visualización del sistema. | Dado que la consulta de alertas devuelve más de 20 registros, cuando el Usuario solicita el siguiente conjunto de datos, entonces el sistema devuelve las siguientes alertas manteniendo activos los filtros aplicados. Dado que el Usuario solicita la sección anterior de datos estando al inicio de la lista, entonces el sistema mantiene el primer conjunto sin cambios. Dado que solicita el siguiente bloque de datos estando en el final de la lista, entonces el sistema mantiene el último conjunto sin realizar nuevas consultas. | EPIC-01 |
| US-04 | Ver Historial de Alertas Resueltas | Como Usuario, quiero consultar las alertas que han vuelto a condiciones normales y se encuentran resueltas, para evaluar el histórico de incidentes ambientales. | Dado que el Usuario solicita el historial de alertas, cuando el sistema procesa la consulta, entonces devuelve únicamente aquellas alertas cuyo estado sea RESOLVED, incluyendo la fecha de ocurrencia y la fecha de resolución de la alerta. Dado que no existen alertas resueltas en el sistema, cuando se realiza la consulta, entonces el sistema devuelve un resultado indicando que no hay registros en el historial. | EPIC-02 |
| US-05 | Distinguir Alertas Activas de Resueltas | Como Usuario, quiero alternar mis consultas entre alertas activas e historial de alertas resueltas, para separar las tareas pendientes de atención de los registros pasados. | Dado que el Usuario desea consultar alertas activas, cuando solicita este subconjunto, entonces el sistema filtra las alertas por estados ACTIVE o ACKNOWLEDGED. Dado que solicita el historial, cuando el sistema procesa la consulta, entonces filtra las alertas por estado RESOLVED y restablece la paginación al primer bloque de datos. | EPIC-02 |
| US-06 | Ver Resumen Diario de Alertas | Como Usuario, quiero obtener un resumen diario del recuento de alertas generadas en la plataforma, para identificar de forma global los patrones de anomalías a lo largo de los días. | Dado que el Usuario solicita el consolidado diario de alertas, cuando el sistema procesa la consulta para los últimos 30 días, entonces devuelve una serie temporal donde cada registro contiene la fecha y el número total de alertas generadas en dicho día. Si en una fecha de la serie no se registraron alertas en el sistema, entonces ese día es devuelto con un valor de recuento de cero. | EPIC-03 |
